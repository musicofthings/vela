Privacy by architecture, not by policy

Your raw data is structurally unreachable.

Every biometric value lives in an AES-256 encrypted vault on your device. The key is derived from your biometric and stored in your phone's Secure Enclave. We can't read it. Not won't — can't.

Servers never see

  • Raw biometric values (HR, HRV, glucose, SpO₂)
  • Timestamps at precision below 1 hour
  • User ID, device ID, any PII
  • Location data of any kind
  • Wearable device identifiers
  • Sleep-stage sequences

Servers may see (with consent)

  • Anonymized ε-DP metric snapshots for AI analysis
  • Aggregate population trends, group size ≥ 1,000
  • Email hash (bcrypt), subscription status, app version
  • Country-level analytics, opt-in only
  • Device class (iOS / Android) for crash reporting
  • Nothing else — this list is exhaustive

What happens when you ask Claude a question

  1. 1Read encrypted vault (Face ID / fingerprint required)
  2. 2Round timestamps to the nearest hour
  3. 3Inject Gaussian DP noise · ε = 0.3, δ = 10⁻⁵
  4. 4Strip user_id → ephemeral per-session token
  5. 5TLS 1.3 to Claude · zero-data-retention
  6. 6Store insight locally · snapshot is never retained

What's inside

Six surfaces, one coherent picture of your health.

P0

Morning Signal

One daily score from HRV, RHR, sleep efficiency, glucose CV, and skin-temp deviation — with a one-sentence Claude explanation. Tap any input to see its raw contribution.

P0

Sleep Intelligence

4-stage sleep classification from PPG + actigraphy via on-device TFLite. ≥82% accuracy vs polysomnography. When CGM is active, nocturnal glucose overlays sleep architecture.

P1

Metabolic Intelligence

Live glucose widget, meal response tracking, TIR / CV / mean. The first wearable platform to natively correlate CGM with HRV and recovery.

P1

VELA Intelligence (Claude)

Conversational coaching over anonymized snapshots via tool use. Proactive insights triggered by statistical patterns, not generic reminders. Max 2 nudges/day.

P2

Anomaly detection

On-device Isolation Forest flags outliers >2 SD from your 30-day baseline. Flag, never alarm — you decide which deserve a closer look.

P3

Genomic + Clinical overlay

Import polygenic risk scores and FHIR R4 records. Pharmacogenomics alerts cross-reference CYP2D6 / CYP2C19 / SLCO1B1 with your medication log.

Two-tier AI architecture

On-device for real time. Claude for reasoning.

Tier 1 · on-device

TFLite · Core ML · ≤10 MB models

  • Sleep staging (LSTM, 4-layer)
  • Anomaly detection (Isolation Forest → ONNX)
  • Glucose 30-min forecast (1D-CNN, causal)
  • HRV feature extractor (NeuroKit2 → TFLite)

Tier 2 · cloud AI

claude-sonnet-4-6 · ZDR · tool use

  • Conversational coaching
  • Weekly digest + suggested experiments
  • Cross-signal reasoning over anonymized snapshots
  • Meal analysis (NLP → glycemic load estimate)
